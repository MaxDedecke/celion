head> Jira Cloud API – Endpunktübersicht & Request-Guide 
Jira Cloud API – Endpunktübersicht & Request-Guide (LLM-optimiert)

========================================================================
1. GRUNDLAGEN
========================================================================

Basis-URL (klassische Site-Integration, z. B. Basic Auth oder 3LO mit Site-URL):

  https://<your-domain>.atlassian.net/rest/api/3/

Basis-URL (API über api.atlassian.com, z. B. reine OAuth 3LO):

  https://api.atlassian.com/ex/jira/<cloudId>/rest/api/3/

Alle nachfolgenden Pfade sind relativ zu /rest/api/3/.

AUTHENTIFIZIERUNG (einfachste Variante für Scripte):

  Authorization: Basic <base64(email:api_token)>
  Accept: application/json
  Content-Type: application/json (bei Bodies)

Generisches Muster für REST-Requests:

  GET    /resource            – Daten lesen
  POST   /resource            – Daten erstellen / Aktionen
  PUT    /resource/{id}       – Ressourcen aktualisieren
  DELETE /resource/{id}       – Ressourcen löschen

Viele Ressourcen sind paginiert:
  Query-Parameter: startAt, maxResults
  Response: { "startAt", "maxResults", "total", "isLast", "values"/"issues"/... }


========================================================================
2. TOP-LEVEL RESSOURCENGRUPPEN
========================================================================

Jira Cloud REST v3 gruppiert Endpunkte grob nach:

- Issues & Suche
- Issue-Komponenten (Kommentare, Worklogs, Attachments, Properties, Votes, Watchers, Links)
- Projekte, Projekt-Properties, -Rollen, -Komponenten, -Versionen, -Avatare
- Felder, Custom Fields, Konfigurationen
- Workflows, Workflow-Schemata, Status, Übergänge
- Nutzer, Gruppen, Berechtigungen, Permission Schemes, Rollen
- Filter, Dashboards, JQL
- Labels, Priorities, Resolutions
- Webhooks, Audit, License Metrics, Myself, Server Info
- Diverse Settings & Admin-Ressourcen

Im Folgenden eine gruppierte Liste der wichtigsten / typischen Endpunkte (REST API v3).

3. ISSUES & ISSUE-NAHE RESSOURCEN
========================================================================

---------------------------------------
3.1 Issues – Kernobjekt
---------------------------------------

Basis: /issue

  POST   /issue
         – Issue erstellen (Body: {"fields": {...}})

  GET    /issue/{issueIdOrKey}
         – Einzelnes Issue abrufen

  PUT    /issue/{issueIdOrKey}
         – Issue aktualisieren (Teil- oder Vollupdate der fields)

  DELETE /issue/{issueIdOrKey}
         – Issue löschen

  POST   /issue/bulk
         – Mehrere Issues in einem Request erstellen

  POST   /issue/archive
         – Mehrere Issues archivieren

  PUT    /issue/unarchive
         – Mehrere Issues aus Archiv holen

  POST   /issue/bulkfetch
         – Mehrere Issues per IDs/Keys in einem Request abrufen

  GET    /issue/createmeta
  GET    /issue/createmeta/{projectIdOrKey}/issuetypes
  GET    /issue/createmeta/{projectIdOrKey}/issuetypes/{issueTypeId}
         – Metadaten zu Feldern/Issuetypes beim Erstellen abrufen

  GET    /issue/limit/report
         – Informationen zu Issue-Beschränkungen (Limits)


---------------------------------------
3.2 Issue-Changelogs & Events
---------------------------------------

  GET    /issue/{issueIdOrKey}/changelog

  POST   /issue/{issueIdOrKey}/changelog/list

11. REQUEST-BEISPIELE
========================================================================

GET – Aktuellen User abrufen:

  GET https://<your-domain>.atlassian.net/rest/api/3/myself
  Authorization: Basic <base64(email:api_token)>
  Accept: application/json

GET – Issues per JQL:

  POST https://<your-domain>.atlassian.net/rest/api/3/search
  Authorization: Basic <...>
  Content-Type: application/json
  Accept: application/json

  Body:
  {
    "jql": "project = ABC ORDER BY created DESC",
    "maxResults": 50,
    "fields": ["summary","status","assignee","created"]
  }

POST – Issue erstellen:

  POST https://<your-domain>.atlassian.net/rest/api/3/issue
  Authorization: Basic <...>
  Content-Type: application/json

  {
    "fields": {
      "project": { "key": "ABC" },
      "summary": "My new issue via REST",
      "issuetype": { "id": "10001" }
    }
  }


========================================================================
12. HINWEISE FÜR LLM-AGENTEN
========================================================================

- Nutze immer vollständige URL: https://<domain>.atlassian.net/rest/api/3/<pfad>
- Authentifiziere dich mit Basic (E-Mail + API-Token) oder OAuth 2.0 Access Token.
- Achte auf Pagination: startAt, maxResults, isLast, total.
- Viele Felder sind dynamisch (customfield_XXXXX) und müssen _nicht_ hart gecodet werden.
- Nutze /field und /issue/createmeta, um herauszufinden, welche Felder existieren.
- JQL /search ist der mächtigste Weg, um Issues in Masse abzurufen.
- Datei-Uploads benötigen X-Atlassian-Token: no-check und multipart/form-data.
- Für Admin-Funktionen (Workflows, Schemes etc.) sind meist globale Admin-Rechte nötig.
- Für wirklich vollständige Details (Schemas, Parameter, Example-Bodies) kann zusätzlich die OpenAPI/Swagger-Datei genutzt werden:
  /rest/api/3/openapi (bzw. die offizielle JSON-Spezifikation)
